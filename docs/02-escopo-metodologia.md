# 2. Escopo e Metodologia

## Escopo Incluído

### Processo Comercial
- Oportunidades
- Negociação
- Cotação
- Aprovação

### Arquitetura
- Modelo de dados
- Objetos padrão
- Objetos customizados

### Automações
- Triggers
- Apex
- Flows
- Batch Jobs

### Integrações
- Sistemas corporativos
- Pricing
- Contratos
- Sistemas legados

### Segurança
- Perfis
- Permissões
- Compartilhamento

## Escopo Não Incluído

### Processos Operacionais Posteriores
- Logística
- Distribuição
- Faturamento

(exceto quando diretamente impactados pela negociação)

---

## Metodologia Utilizada

### Fontes de Informação

#### Workshops
Sessões com:
- Negócio
- Arquitetura
- Desenvolvimento
- Sustentação

#### Análise Técnica
Avaliação de:
- Metadata
- Código Apex
- Flows
- Objetos
- Configurações

#### Entrevistas
Levantamento com stakeholders.

#### Documentação Existente
- Materiais funcionais
- Diagramas
- Artefatos de arquitetura

### Framework de Avaliação

A análise foi estruturada em cinco dimensões:

| Dimensão | Objetivo |
|----------|----------|
| Arquitetura | Avaliar desacoplamento e modularidade |
| Dados | Avaliar qualidade e governança |
| Automação | Avaliar complexidade operacional |
| Integração | Avaliar dependências externas |
| Governança | Avaliar capacidade de evolução |
